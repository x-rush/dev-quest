# 工作流启动指南

> **使用说明**: 这是启动Claude Code自动化工作流的详细指南，包含从规划到执行的全流程。

## 🚀 快速启动流程

### 步骤1: 环境准备
```bash
# 1. 确保你在项目根目录
pwd  # 应该显示你的项目路径

# 2. 复制模板文件到项目
cp -r claude-workflow-template/template-docs/ ./
cp -r claude-workflow-template/scripts/ ./

# 3. 验证文件复制成功
ls -la PROJECT_PLAN.md TASK_DEFINITIONS.md QUALITY_STANDARDS.md
```

### 步骤2: 开始规划对话
```bash
# 启动Claude Code
claude "我想要为 [你的项目描述] 创建自动化执行计划，请根据 PROJECT_PLAN_TEMPLATE.md 帮我制定详细的项目规划"
```

### 步骤3: 规划对话模板
```
人类: 我想要为 Dev Quest 项目生成完整的Go后端学习模块文档
Claude: 我来帮你制定详细的执行计划。首先需要明确几个问题：

1. **项目目标**:
   - 这个Go后端学习模块具体要包含哪些技术内容？
   - 目标读者是谁？（比如：有PHP基础的开发者）
   - 预期达到什么效果？（完整的文档体系，实战案例等）

2. **技术范围**:
   - Go版本要求？（建议Go 1.21+）
   - 涵盖的框架？（Gin, Echo, 标准库等）
   - 包含哪些主题？（基础语法、Web开发、数据库、微服务等）

3. **质量标准**:
   - 文档详细程度？
   - 代码示例要求？
   - 实战案例复杂度？

4. **时间安排**:
   - 希望什么时候完成？
   - 是否需要分阶段交付？

请告诉我这些信息，我会为你制定详细的执行计划。
```

### 步骤4: 确认规划
```
Claude: 根据你的需求，我已经制定了详细的项目规划，主要包括：

**项目结构**:
- 基础知识模块 (5个文档)
- Web开发模块 (8个文档)
- 数据库模块 (6个文档)
- 微服务模块 (4个文档)
- 实战项目模块 (3个文档)

**质量标准**:
- 每个文档包含完整的代码示例
- 所有代码经过可执行性验证
- 文档格式统一，包含实战案例
- 使用2025年最新的Go生态最佳实践

**执行阶段**:
- 阶段1: 基础设施搭建 (2小时)
- 阶段2: 核心内容生成 (8小时)
- 阶段3: 质量验证和优化 (2小时)

**风险评估**:
- 代码示例可执行性验证：中等风险
- 文档格式一致性：低风险
- 技术内容准确性：低风险

请确认这个规划是否符合你的预期，或者有什么需要调整的地方？
```

### 步骤5: 细化任务定义
```
人类: 规划很好，请帮我生成详细的任务定义
Claude: 好的，我来根据规划生成具体的任务定义。我会按照 TASK_DEFINITION_TEMPLATE.md 的格式，为每个子任务定义：
- 具体执行步骤
- 验证标准
- 异常处理方案
- 完成标志

正在生成 TASK_DEFINITIONS.md...

[生成任务定义文件]

任务定义已完成！包含了26个具体任务，每个任务都有明确的执行标准和验证方法。请检查是否需要调整。
```

### 步骤6: 质量标准确认
```
人类: 任务定义很详细，质量标准呢？
Claude: 我已经根据 QUALITY_STANDARDS_TEMPLATE.md 生成了这个项目的具体质量标准，包括：

**技术准确性** (40%):
- 代码示例语法正确，可执行
- Go语言特性描述准确
- 最佳实践符合当前版本

**内容完整性** (25%):
- 每个主题覆盖完整的学习路径
- 从基础概念到实际应用
- 包含常见问题和解决方案

**实用性** (20%):
- 提供真实项目场景
- 包含完整的配置步骤
- 给出性能优化建议

**前沿性** (15%):
- 使用Go 1.21+最新特性
- 包含2025年生态系统工具
- 展示现代开发模式

请确认这些质量标准是否合适？
```

## 🔄 执行监控指南

### 实时监控方法
```bash
# 方法1: 查看执行状态文件
tail -f EXECUTION_STATUS.json

# 方法2: 查看执行日志
tail -f EXECUTION_LOG.md

# 方法3: 查看当前任务
grep "current_task" EXECUTION_STATUS.json

# 方法4: 监控文件变化
watch -n 5 'find . -name "*.md" -mmin -10 | head -5'
```

### 进度检查命令
```bash
# 检查总体进度
grep -A 5 "execution_info" EXECUTION_STATUS.json

# 检查任务状态分布
grep -c "COMPLETED" EXECUTION_STATUS.json
grep -c "FAILED" EXECUTION_STATUS.json
grep -c "IN_PROGRESS" EXECUTION_STATUS.json

# 检查质量指标
grep "quality_score" EXECUTION_STATUS.json
```

## ⚠️ 常见问题和解决方案

### 问题1: 执行中断
```markdown
**症状**: Claude Code执行突然停止
**原因**: 可能是网络问题、系统资源不足、或API限制
**解决方案**:
1. 检查网络连接
2. 重启Claude Code
3. 从断点恢复: claude "根据 EXECUTION_STATUS.json 从中断点继续执行"
```

### 问题2: 任务失败
```markdown
**症状**: 某个任务执行失败，显示错误信息
**原因**: 权限问题、磁盘空间、或内容验证失败
**解决方案**:
1. 查看ERROR_REPORT.md了解具体错误
2. 根据错误类型进行人工干预
3. 修复问题后: claude "继续执行失败的任务 [任务ID]"
```

### 问题3: 质量不达标
```markdown
**症状**: 生成的内容质量不符合标准
**原因**: 质量标准定义不清晰或验证规则有误
**解决方案**:
1. 检查QUALITY_STANDARDS.md定义
2. 调整质量标准或验证规则
3. 重新执行质量检查: claude "重新验证已生成的内容质量"
```

### 问题4: 执行缓慢
```markdown
**症状**: 执行速度比预期慢很多
**原因**: 系统资源不足或任务定义过于复杂
**解决方案**:
1. 检查系统资源使用情况
2. 优化任务定义，减少不必要的验证
3. 调整并行执行参数
```
